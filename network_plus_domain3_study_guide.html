EOS: Vendor stops providing technical support and updates

Practice Quiz - Domain 3.0

Test your knowledge with realistic exam-style questions

1. A network administrator needs to implement a monitoring solution that can collect performance data from multiple vendors' network devices. Which protocol would be MOST appropriate?
A) Telnet
B) SNMP
C) SSH
D) HTTP
Correct Answer: B) SNMP
SNMP (Simple Network Management Protocol) is the standard protocol for network monitoring and management across multiple vendor platforms. It provides a standardized way to collect performance metrics, configuration data, and receive alerts from network devices.
2. An organization requires a disaster recovery solution with an RTO of 2 hours and minimal data loss. Which type of DR site would be MOST appropriate?
A) Cold site
B) Warm site
C) Hot site
D) Cloud backup
Correct Answer: C) Hot site
A hot site provides the fastest recovery time (minutes to hours) and minimal data loss through real-time replication. It's fully equipped and ready to take over operations immediately, making it ideal for organizations with strict RTO requirements.
3. Which DHCP process step involves the client broadcasting a request for an IP address?
A) Offer
B) Request
C) Discover
D) Acknowledge
Correct Answer: C) Discover
The DHCP process follows DORA: Discover (client broadcasts request), Offer (server responds with available IP), Request (client requests the offered IP), Acknowledge (server confirms assignment).
4. In an IPv6 network using SLAAC, what mechanism ensures that automatically configured addresses are unique?
A) Router Advertisement (RA)
B) Neighbor Discovery (ND)
C) Duplicate Address Detection (DAD)
D) DHCPv6
Correct Answer: C) Duplicate Address Detection (DAD)
DAD is the process used in IPv6 SLAAC to verify that an automatically configured address is unique on the network segment. It sends Neighbor Solicitation messages to check if any other device is using the same address.
5. A network administrator wants to ensure that management traffic is available even when the production network fails. Which approach should be implemented?
A) In-band management
B) Out-of-band management
C) SNMP monitoring
D) SSH access
Correct Answer: B) Out-of-band management
Out-of-band management uses a separate network path for management traffic, ensuring that device management remains available even when the production network fails. This typically involves console servers or dedicated management networks.
6. Which DNS record type is used to specify the mail server for a domain?
A) A record
B) CNAME record
C) MX record
D) PTR record
Correct Answer: C) MX record
MX (Mail Exchange) records specify the mail server responsible for accepting email messages on behalf of a domain. They include a priority value to support multiple mail servers.
7. In a split tunnel VPN configuration, which traffic is routed through the VPN tunnel?
A) All traffic
B) Only corporate network traffic
C) Only internet traffic
D) Only encrypted traffic
Correct Answer: B) Only corporate network traffic
In split tunneling, only traffic destined for the corporate network is routed through the VPN tunnel, while internet traffic goes directly from the client to the internet. This reduces bandwidth usage on the corporate connection.
8. Which metric defines the maximum acceptable amount of data loss in a disaster recovery plan?
A) RTO (Recovery Time Objective)
B) RPO (Recovery Point Objective)
C) MTTR (Mean Time to Repair)
D) MTBF (Mean Time Between Failures)
Correct Answer: B) RPO (Recovery Point Objective)
RPO defines the maximum acceptable amount of data loss measured in time. For example, an RPO of 1 hour means the organization can tolerate losing up to 1 hour of data in a disaster scenario.

3.2 Network Monitoring Technologies

Implement comprehensive network monitoring and performance management

Simple Network Management Protocol (SNMP)

SNMP Components

  • SNMP Manager: Monitoring system that collects data
  • SNMP Agent: Software on monitored devices
  • Management Information Base (MIB): Database of manageable objects

SNMP Versions

  • SNMPv1: Original version, basic functionality, security via community strings
  • SNMPv2c: Improved performance, bulk transfers, community-based security
  • SNMPv3: Enhanced security with authentication and encryption
snmp-server community public ro snmp-server community private rw snmp-server host 192.168.1.100 version 2c public

SNMP Operations

  • GET: Retrieve specific object values
  • SET: Modify object values
  • GET-NEXT: Retrieve next object in MIB tree
  • TRAP: Unsolicited alert from agent to manager

Flow Data and Packet Capture

Flow Monitoring Technologies

  • NetFlow: Cisco's flow monitoring protocol
  • sFlow: Statistical packet sampling
  • IPFIX: Internet Protocol Flow Information Export (standard)

Flow data provides insights into:

  • Traffic patterns and volume
  • Application usage
  • Security threat detection
  • Bandwidth utilization

Packet Capture Methods

  • Port Mirroring (SPAN): Copy traffic from one port to analyzer
  • Network TAPs: Physical devices that split signals
  • Software Captures: Wireshark, tcpdump, tshark

Baseline Metrics and Alerting

Key Performance Metrics

  • Bandwidth Utilization: Current vs. available capacity
  • Latency: Round-trip time for packets
  • Packet Loss: Percentage of lost packets
  • Jitter: Variation in packet delivery times
  • Error Rates: CRC errors, collisions, runts

Anomaly Detection

Automated alerting when metrics exceed baseline thresholds:

  • CPU utilization > 80% for 5 minutes
  • Interface utilization > 90%
  • Memory usage > 85%
  • Temperature above device limits

Log Aggregation and SIEM

Syslog Protocol

Standard protocol for collecting and forwarding log messages:

  • Facility: Source of the message (mail, kernel, security)
  • Severity: Importance level (0=Emergency to 7=Debug)
  • Timestamp: When the event occurred
  • Hostname: Source device
  • Message: Actual log content
logging host 192.168.1.10 logging trap informational logging facility local0

Security Information and Event Management (SIEM)

Centralized platform that:

  • Collects logs from multiple sources
  • Correlates events across systems
  • Provides real-time alerting
  • Supports forensic analysis
  • Generates compliance reports

3.3 Disaster Recovery Concepts

Design and implement comprehensive disaster recovery strategies

Disaster Recovery Metrics

Recovery Point Objective (RPO)

Maximum acceptable amount of data loss measured in time:

  • Zero RPO: No data loss acceptable (real-time replication)
  • 1 Hour RPO: Maximum 1 hour of data loss acceptable
  • 24 Hour RPO: Daily backups acceptable

Recovery Time Objective (RTO)

Maximum acceptable downtime before operations must be restored:

  • Critical Systems: RTO = 1 hour
  • Important Systems: RTO = 4 hours
  • Non-Critical Systems: RTO = 24-72 hours

Mean Time Metrics

  • Mean Time to Repair (MTTR): Average time to fix a failed component
  • Mean Time Between Failures (MTBF): Average operational time between failures
Formula: Availability = MTBF / (MTBF + MTTR)

Disaster Recovery Site Types

Cold Site

  • Description: Backup facility with basic infrastructure (power, cooling, network)
  • RTO: Days to weeks
  • Cost: Lowest
  • Use Case: Non-critical systems, small organizations

Warm Site

  • Description: Partially equipped facility with some hardware and software
  • RTO: Hours to days
  • Cost: Medium
  • Use Case: Balanced cost and recovery time requirements

Hot Site

  • Description: Fully equipped duplicate facility with real-time data replication
  • RTO: Minutes to hours
  • Cost: Highest
  • Use Case: Mission-critical systems requiring minimal downtime

High Availability Approaches

Active-Active Configuration

  • Multiple systems handle traffic simultaneously
  • Load is distributed across all active systems
  • If one system fails, others continue operating
  • Provides both redundancy and increased capacity
Load Balancer

Server 1 (Active)
Server 2 (Active)

Active-Passive Configuration

  • One system handles all traffic (active)
  • Backup system remains on standby (passive)
  • Automatic failover when active system fails
  • Standby system must be capable of handling full load
Primary Server (Active)

Backup Server (Passive)

Disaster Recovery Testing

Tabletop Exercises

  • Discussion-based testing of DR procedures
  • No actual system disruption
  • Identifies gaps in procedures and communication
  • Cost-effective and low-risk

Validation Tests

  • Backup Validation: Verify backup integrity and restorability
  • Failover Testing: Test automatic switching to backup systems
  • Full DR Test: Complete system failover to alternate site

3.4 IPv4 and IPv6 Network Services

Configure and manage essential network services

Dynamic Host Configuration Protocol (DHCP)

DHCP Components

  • DHCP Server: Assigns IP addresses and configuration
  • DHCP Client: Requests IP configuration
  • DHCP Relay Agent: Forwards DHCP requests across subnets

DHCP Process (DORA)

  1. Discover: Client broadcasts request for IP address
  2. Offer: Server responds with available IP address
  3. Request: Client requests the offered address
  4. Acknowledge: Server confirms the assignment
ip dhcp pool LAN_POOL network 192.168.1.0 255.255.255.0 default-router 192.168.1.1 dns-server 8.8.8.8 8.8.4.4 lease 7

DHCP Options

  • Reservations: Static IP assignments based on MAC address
  • Scope: Range of IP addresses available for assignment
  • Lease Time: Duration of IP address assignment
  • Exclusions: IP addresses not available for dynamic assignment
  • Relay/IP Helper: Forward DHCP across router boundaries

IPv6 Address Configuration

Stateless Address Autoconfiguration (SLAAC)

Automatic IPv6 address configuration using Router Advertisements:

  • Router sends periodic Router Advertisements (RA)
  • Contains network prefix and configuration flags
  • Host generates Interface ID using EUI-64 or random
  • Duplicate Address Detection (DAD) ensures uniqueness
interface GigabitEthernet0/0 ipv6 address 2001:db8:1::1/64 ipv6 nd prefix default no-autoconfig ipv6 dhcp server DHCPv6_POOL

DHCPv6 vs SLAAC

Aspect SLAAC DHCPv6
Address Assignment Automatic (prefix + interface ID) Server-managed pool
DNS Configuration Router Advertisement (RDNSS) DHCPv6 options
Management Overhead Low Medium
Address Tracking None Full logging

Domain Name System (DNS)

DNS Record Types

  • A (Address): Maps hostname to IPv4 address
  • AAAA: Maps hostname to IPv6 address
  • CNAME (Canonical Name): Alias for another domain name
  • MX (Mail Exchange): Mail server for domain
  • NS (Name Server): Authoritative name server for domain
  • PTR (Pointer): Reverse DNS lookup (IP to hostname)
  • TXT (Text): Arbitrary text data

DNS Zone Types

  • Forward Zone: Resolves hostnames to IP addresses
  • Reverse Zone: Resolves IP addresses to hostnames
  • Primary Zone: Master copy of zone data
  • Secondary Zone: Read-only copy from primary

DNS Security Extensions (DNSSEC)

Provides authentication and data integrity:

  • Digital signatures for DNS records
  • Chain of trust from root to authoritative servers
  • Prevents DNS spoofing and cache poisoning
  • Uses public key cryptography

DNS over HTTPS (DoH) and DNS over TLS (DoT)

  • DoH: DNS queries over HTTPS (port 443)
  • DoT: DNS queries over TLS (port 853)
  • Both provide encryption and privacy
  • Prevent eavesdropping on DNS queries

Time Protocols

Network Time Protocol (NTP)

  • Synchronizes system clocks across network
  • Hierarchical structure with stratum levels
  • Stratum 0: Reference clocks (GPS, atomic)
  • Stratum 1: Primary time servers
  • Uses UDP port 123
ntp server 0.pool.ntp.org ntp server 1.pool.ntp.org ntp authenticate ntp trusted-key 1

Precision Time Protocol (PTP)

  • Higher precision than NTP (nanosecond accuracy)
  • Used in industrial and financial applications
  • Master-slave hierarchy
  • Hardware timestamping support

Network Time Security (NTS)

  • Security extension for NTP
  • Provides authentication and integrity
  • Prevents time spoofing attacks
  • Uses TLS for secure key exchange

3.5 Network Access and Management Methods

Configure secure remote access and network management

Virtual Private Networks (VPNs)

Site-to-Site VPN

  • Connects entire networks across public internet
  • Gateway-to-gateway connection
  • Transparent to end users
  • Common protocols: IPSec, L2TP, GRE
Site A Gateway
Internet
Site B Gateway

Client-to-Site VPN

  • Individual devices connect to corporate network
  • Remote worker access solution
  • Client software required
  • Two main types: Clientless and Client-based

Clientless VPN:

  • Web browser-based access
  • No software installation required
  • Limited application support
  • SSL/TLS tunnel

Client-based VPN:

  • Dedicated VPN client software
  • Full network layer access
  • Supports all applications
  • More secure and feature-rich

Split Tunnel vs Full Tunnel

Aspect Split Tunnel Full Tunnel
Internet Traffic Direct to internet Through corporate gateway
Corporate Traffic Through VPN tunnel Through VPN tunnel
Bandwidth Usage Lower Higher
Security Lower (some traffic unmonitored) Higher (all traffic monitored)
Performance Better for internet access May be slower

Network Management Connection Methods

Secure Shell (SSH)

  • Encrypted command-line access
  • Replaces insecure Telnet
  • Uses TCP port 22
  • Supports public key authentication
ssh -l admin 192.168.1.1 ssh -i ~/.ssh/id_rsa admin@switch01

Graphical User Interface (GUI)

  • Web-based management interfaces
  • Point-and-click configuration
  • Usually HTTPS-based (port 443)
  • Good for initial setup and basic management

Application Programming Interface (API)

  • Programmatic access to device functions
  • RESTful APIs common in modern devices
  • JSON or XML data formats
  • Enables automation and scripting
curl -X GET "https://switch01/api/v1/interfaces" \ -H "Authorization: Bearer token123" \ -H "Content-Type: application/json"

Console Access

  • Direct physical connection to device
  • Serial cable (RS-232) or USB
  • Out-of-band management
  • Works even when network is down

Jump Box/Host and Management Methods

Jump Box/Bastion Host

  • Secure intermediary system for network access
  • Single point of entry to secure network segments
  • Detailed logging and auditing
  • Multi-factor authentication
Administrator
Jump Box
Internal Network

In-band vs Out-of-band Management

In-band Management:

  • Uses production network for management traffic
  • SSH, SNMP, HTTP/HTTPS over main network
  • Lower cost, easier to implement
  • Management unavailable if network fails

Out-of-band Management:

  • Separate network for management traffic
  • Console servers, dedicated management network
  • Available even when production network fails
  • Higher cost but more reliable
