Domain 4 Overview: Network Security Fundamentals

Domain Weight and Importance

Domain 4 represents 14% of the N10-009 exam and covers critical security concepts that every network professional must understand. This domain focuses on protecting network infrastructure from various threats and implementing proper security controls.

4.1 Basic Network Security Concepts

The CIA Triad

Confidentiality: Ensuring data is accessible only to authorized users through encryption and access controls.

Integrity: Maintaining data accuracy and preventing unauthorized modifications.

Availability: Ensuring systems and data are accessible when needed by authorized users.

Risk Management Terminology

Risk: The potential for loss or damage when a threat exploits a vulnerability.

Threat: Any potential danger that could exploit a vulnerability.

Vulnerability: A weakness in a system that could be exploited.

Exploit: A specific attack that takes advantage of a vulnerability.

Logical Security Concepts

Encryption Fundamentals

Data in Transit vs Data at Rest

Data in Transit: Information being transmitted across networks. Protected using protocols like TLS/SSL, IPSec, and VPN tunnels.

Data at Rest: Information stored on devices or systems. Protected using full-disk encryption, file-level encryption, and database encryption.

Public Key Infrastructure (PKI)

PKI Components

Certificate Authority (CA): Trusted entity that issues and manages digital certificates.

Registration Authority (RA): Verifies certificate requests before forwarding to CA.

Digital Certificates: Electronic credentials that bind public keys to identities.

Public/Private Key Pairs: Cryptographic keys used for encryption and digital signatures.

# Example: Generating a self-signed certificate openssl req -x509 -newkey rsa:4096 -keyout private.key -out certificate.crt -days 365

Certificate Types

Self-Signed Certificates: Created and signed by the same entity, suitable for internal testing but not trusted by default.

CA-Signed Certificates: Issued by trusted Certificate Authorities, automatically trusted by most systems.

Wildcard Certificates: Cover multiple subdomains (*.example.com).

Extended Validation (EV): Highest level of validation, shows organization name in browser.

Identity and Access Management (IAM)

Authentication Methods

Single Factor: Password only

Two Factor (2FA): Something you know + something you have

Multi-Factor (MFA): Multiple authentication factors

Biometric: Fingerprint, facial recognition, retina scan

Authentication Protocols

RADIUS: Remote Authentication Dial-in User Service

LDAP: Lightweight Directory Access Protocol

SAML: Security Assertion Markup Language

TACACS+: Terminal Access Controller Access Control System Plus

Authorization Concepts

Least Privilege: Users get minimum necessary access

Role-Based Access Control (RBAC): Access based on job functions

Time-Based Authentication: Access restrictions by time

Geofencing: Location-based access controls

Single Sign-On (SSO) Technologies

SSO Benefits and Implementation

SSO allows users to authenticate once and access multiple systems without re-entering credentials.

Technology Use Case Protocol Security Level

Environmental Security

Modern data centers implement multiple layers of physical security:

  • Perimeter Security: Fencing, barriers, vehicle controls

  • Floor Access: Biometric scanners, proximity cards
  • Rack Security: Locked cabinets, individual server locks

Deception Technologies

Honeypots and Honeynets

Honeypot: A decoy system designed to attract and detect attackers by simulating vulnerable services or data.

Honeynet: A network of honeypots that creates a more realistic environment for studying attack patterns.

Types of Honeypots:

  • Low-Interaction: Simulated services, safer but less detailed
  • High-Interaction: Real systems, higher risk but better intelligence
  • Production: Deployed in live environments
  • Research: Used for threat intelligence gathering
# Example: Simple honeypot configuration # Create a fake SSH service on non-standard port nc -l -p 2222 -e /bin/bash # Log all connection attempts for analysis
Physical security is often the weakest link in organizational security. A determined attacker with physical access can bypass most logical security controls.

Network Attack Types and Techniques

Denial of Service Attacks

DoS (Denial of Service)

Objective: Overwhelm a single target from one source

Methods:

  • SYN Flood: Exploit TCP handshake process
  • UDP Flood: Overwhelm with UDP packets
  • ICMP Flood: Ping flood attacks
  • Application Layer: Target specific applications

Impact: Service unavailability, resource exhaustion

DDoS (Distributed DoS)

Objective: Coordinated attack from multiple sources

Botnet Usage: Networks of compromised devices

Attack Vectors:

  • Volumetric: High bandwidth consumption
  • Protocol: Exploit protocol weaknesses
  • Application: Target application logic

Amplification: DNS, NTP, SNMP reflection attacks

Layer 2 Attacks

VLAN Hopping

Switch Spoofing: Negotiate trunk mode on access port

Double Tagging: Exploit native VLAN processing

Prevention:

  • Disable unused ports
  • Configure access ports explicitly
  • Use dedicated VLAN for trunk native
  • Implement VLAN ACLs

MAC Flooding

CAM Table Overflow: Exhaust switch memory with fake MAC addresses

Failopen Mode: Switch acts as hub when table full

Prevention:

  • Port security configuration
  • MAC address limiting
  • Dynamic learning limits
  • Monitoring CAM table usage

ARP Poisoning/Spoofing

Man-in-the-Middle: Intercept traffic between devices

Gateway Impersonation: Redirect default route traffic

Detection:

  • ARP table monitoring
  • Duplicate MAC detection
  • Network baseline comparison
# Detect ARP poisoning arp -a | grep "00:11:22:33:44:55" # Check for duplicate MAC addresses

DNS Attacks

DNS Poisoning

Cache Poisoning: Corrupt DNS resolver cache

Response Injection: Insert malicious DNS responses

Prevention:

  • DNSSEC implementation
  • DNS over HTTPS (DoH)
  • DNS over TLS (DoT)
  • Secure DNS servers

DNS Spoofing

False Responses: Send fake DNS replies

Traffic Redirection: Route users to malicious sites

Pharming: Large-scale DNS redirection

Wireless Attacks

Evil Twin

Rogue Access Point: Fake AP with legitimate SSID

Credential Harvesting: Capture login credentials

Man-in-the-Middle: Intercept wireless traffic

Prevention:

  • Certificate-based authentication
  • Wireless intrusion detection
  • User education

Rogue Devices

Rogue DHCP: Unauthorized DHCP servers

Rogue AP: Unauthorized access points

Detection Methods:

  • Network scanning
  • DHCP snooping
  • Wireless monitoring
  • Network access control

Social Engineering

Phishing

Email-based: Malicious emails requesting credentials

Spear Phishing: Targeted attacks on specific individuals

Whaling: Target high-value executives

Physical Social Engineering

Tailgating: Following authorized personnel

Shoulder Surfing: Observing credentials being entered

Dumpster Diving: Searching discarded materials

Security Features and Defense Techniques

Device Hardening

Basic Hardening Steps

Disable Unused Services: Reduce attack surface

Change Default Passwords: Eliminate known credentials

Update Firmware: Apply security patches

Configure Logging: Enable audit trails

# Example: Router hardening commands no ip http server no ip http secure-server service password-encryption logging buffered 16384

Port Security

Disable Unused Ports: Shutdown inactive interfaces

MAC Address Limiting: Restrict allowed devices

Violation Actions: Shutdown, restrict, or protect

# Cisco switch port security interface fa0/1 switchport mode access switchport port-security switchport port-security max 2 switchport port-security violation shutdown

Network Access Control (NAC)

802.1X Port-Based Authentication

Components:

  • Supplicant: Client device requesting access
  • Authenticator: Network device (switch/AP) controlling access
  • Authentication Server: RADIUS server validating credentials

EAP Methods:

  • EAP-TLS: Certificate-based, highest security
  • EAP-PEAP: Protected EAP with TLS tunnel
  • EAP-TTLS: Tunneled TLS for legacy authentication
  • EAP-FAST: Flexible Authentication via Secure Tunneling

NAC Implementation

Pre-admission Control: Device posture assessment

Policy Enforcement: VLAN assignment, ACL application

Guest Access: Captive portal, temporary credentials

BYOD Support: Device registration and profiling

MAC Filtering

Allow Lists: Permit only known devices

Deny Lists: Block specific devices

Limitations: Easy to bypass with spoofing

Best Practice: Combine with other methods

Access Control Lists (ACLs)

ACL Types and Implementation

Standard ACLs (1-99, 1300-1999)

Filter based on source IP address only

# Example: Block specific network access-list 10 deny 192.168.100.0 0.0.0.255 access-list 10 permit any interface fa0/1 ip access-group 10 in

Extended ACLs (100-199, 2000-2699)

Filter based on source, destination, protocol, and port

# Example: Block HTTP to specific server access-list 100 deny tcp any host 10.1.1.100 eq 80 access-list 100 permit ip any any interface fa0/1 ip access-group 100 in

Named ACLs

More flexible and easier to manage

# Example: Named ACL ip access-list extended BLOCK_SOCIAL_MEDIA deny tcp any host 157.240.1.1 eq 80 deny tcp any host 157.240.1.1 eq 443 permit ip any any interface fa0/1 ip access-group BLOCK_SOCIAL_MEDIA in

Content and URL Filtering

URL Filtering

Category-based: Block by content categories

Reputation-based: Block known malicious sites

Custom Lists: Organization-specific blocks

Time-based: Restrict access by schedule

Content Filtering

Deep Packet Inspection: Analyze packet contents

Application Control: Block specific applications

Data Loss Prevention: Prevent sensitive data exfiltration

Malware Detection: Scan for malicious content

Network Zones and Segmentation

Trust Zones

Trusted Zone: Internal corporate network with high trust level

Untrusted Zone: External networks (Internet) with no trust

DMZ/Screened Subnet: Semi-trusted zone for public services

Zone Type Trust Level Typical Services Security Controls
Trusted (Internal) High File servers, databases Internal firewalls, IPS
DMZ Medium Web servers, email Firewalls, monitoring
Untrusted (Internet) None Public web Perimeter firewall

Key Management

Cryptographic Key Lifecycle

  1. Generation: Create keys using secure random number generators
  2. Distribution: Securely deliver keys to authorized parties
  3. Storage: Protect keys using hardware security modules (HSMs)
  4. Usage: Apply keys for encryption/decryption operations
  5. Rotation: Regularly replace keys to limit exposure
  6. Revocation: Invalidate compromised or expired keys
  7. Destruction: Securely delete keys when no longer needed

Audits and Regulatory Compliance

Data Locality Requirements

Geographic Data Restrictions

Data Sovereignty: Legal requirements for data to remain within specific geographical boundaries

Cross-Border Transfers: Restrictions on moving data between countries

Cloud Considerations: Ensuring cloud providers store data in compliant locations

Network Implications: Routing and storage decisions based on regulations

Major Compliance Frameworks

PCI DSS (Payment Card Industry Data Security Standard)

Scope: Organizations handling credit card data

Requirements:

  • Build and maintain secure networks
  • Protect cardholder data
  • Maintain vulnerability management
  • Implement strong access controls
  • Regular monitoring and testing
  • Maintain information security policy

Network Requirements: Firewalls, encryption, access controls

GDPR (General Data Protection Regulation)

Scope: EU residents' personal data protection

Key Principles:

  • Lawfulness, fairness, transparency
  • Purpose limitation
  • Data minimization
  • Accuracy
  • Storage limitation
  • Integrity and confidentiality
  • Accountability

Network Impact: Encryption, access logging, data anonymization

Network Segmentation for Compliance

Specialized Network Segments

IoT (Internet of Things) & IIoT (Industrial IoT)

Security Challenges:

  • Limited processing power for security
  • Default credentials often unchanged
  • Infrequent security updates
  • Large attack surface

Segmentation Strategy:

  • Dedicated IoT VLAN
  • Firewall rules restricting communication
  • Network access control (NAC)
  • Regular monitoring and traffic analysis

SCADA/ICS/OT (Operational Technology)

Critical Infrastructure Protection:

  • Air-gapped networks when possible
  • Strict access controls
  • Protocol-specific firewalls
  • Continuous monitoring

Network Protocols: Modbus, DNP3, IEC 61850, PROFINET

Security Considerations: Legacy systems, real-time requirements, safety implications

Guest Networks

Isolation Requirements:

  • Complete separation from corporate network
  • Limited internet access
  • Captive portal authentication
  • Bandwidth limitations

Implementation: Separate VLAN, firewall rules, web filtering

BYOD (Bring Your Own Device)

Security Challenges:

  • Unmanaged devices
  • Mixed operating systems
  • Personal vs corporate data
  • Compliance requirements

Solutions: Mobile device management (MDM), certificate-based authentication, application containerization

Audit Requirements

Network Security Auditing

Log Management:

  • Centralized logging (SIEM)
  • Log retention policies
  • Log integrity protection
  • Regular log analysis

Access Monitoring:

  • User activity tracking
  • Privilege escalation detection
  • Failed login attempts
  • Administrative actions

Network Traffic Analysis:

  • Baseline traffic patterns
  • Anomaly detection
  • Data exfiltration monitoring
  • Protocol analysis
Compliance is not just about meeting minimum requirements‚Äîit's about implementing best practices that protect the organization and its stakeholders.

Lab 1: Setting Up a Basic PKI Infrastructure

Objective: Create a Certificate Authority and issue certificates for network authentication.

Duration: 2-3 hours

Prerequisites: Virtual machines with Windows Server or Linux

Step-by-Step Instructions:

  1. Install and configure OpenSSL or Windows Certificate Services
  2. Create a Root Certificate Authority (CA)
  3. Generate and configure intermediate CA certificates
  4. Create user certificates for authentication
  5. Configure a web server to require client certificates
  6. Test certificate-based authentication
  7. Implement certificate revocation lists (CRL)
  8. Configure automatic certificate enrollment
# Example OpenSSL commands for CA setup # Generate CA private key openssl genrsa -aes256 -out ca-private-key.pem 4096 # Create CA certificate openssl req -new -x509 -days 3650 -key ca-private-key.pem -sha256 -out ca-certificate.pem # Generate server certificate signing request openssl req -newkey rsa:4096 -nodes -keyout server-private-key.pem -out server.csr # Sign server certificate with CA openssl x509 -req -in server.csr -CA ca-certificate.pem -CAkey ca-private-key.pem -CAcreateserial -out server-certificate.pem -days 365 -sha256
Remember to secure the CA private key properly - it's the root of trust for your entire PKI infrastructure.

Lab Safety and Best Practices

  • Isolated Environment: Always use isolated lab networks to prevent impact on production systems
  • Documentation: Document all configurations, findings, and lessons learned
  • Legal Considerations: Only perform penetration testing on systems you own or have explicit permission to test
  • Backup and Recovery: Always have a way to restore systems to a known good state
  • Collaboration: Work with peers to share knowledge and validate findings
These lab exercises are designed to provide hands-on experience with real-world security scenarios. Take time to understand not just the "how" but also the "why" behind each configuration and security control.
