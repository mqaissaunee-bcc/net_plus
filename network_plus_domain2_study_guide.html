Domain 2.0: Network Implementation Overview

What You'll Learn in This Domain

Domain 2.0 focuses on the practical implementation of network technologies. This is where theory meets practice, covering how to actually deploy, configure, and manage network infrastructure.

Core Learning Objectives:

  • 2.1 Routing Technologies: Static/dynamic routing, NAT, FHRP, and routing protocols
  • 2.2 Switching Technologies: VLANs, spanning tree, interface configuration
  • 2.3 Wireless Technologies: Channel configuration, encryption, authentication
  • 2.4 Physical Installations: Rack design, power, environmental factors
Exam Tip: Domain 2.0 heavily emphasizes hands-on skills. Practice configuring actual network devices or use network simulators to reinforce theoretical knowledge.

2.1 Routing Technologies

Static vs. Dynamic Routing

Static Routing

Manually configured routes that don't change unless administratively modified.

  • Predictable and secure - routes don't change automatically
  • Low resource usage - no routing protocol overhead
  • Best for small networks or specific route requirements
  • Requires manual updates when topology changes
# Example static route configuration ip route 192.168.2.0 255.255.255.0 10.1.1.1 # Destination network | Subnet mask | Next hop

Dynamic Routing Protocols

Protocol Type Metric Use Case
BGP Exterior Gateway (EGP) AS Path, Policy Internet routing between ISPs
EIGRP Advanced Distance Vector Composite (BW, Delay) Cisco networks, fast convergence
OSPF Link State Cost (based on bandwidth) Large enterprise networks
Route Selection Process

When multiple routes exist to the same destination, routers use the following hierarchy:

1. Administrative Distance (AD)

  • Connected: 0 (highest priority)
  • Static: 1
  • EIGRP: 90
  • OSPF: 110
  • RIP: 120 (lowest priority)

2. Prefix Length (Longest Match)

More specific routes (longer subnet masks) are preferred.

/24 network (255.255.255.0) preferred over /16 network (255.255.0.0) for the same destination

3. Metric

Used when routes have the same AD and prefix length. Lower metric = better route.

Network Address Translation (NAT)

NAT Types

  • Static NAT: One-to-one mapping between private and public IP
  • Dynamic NAT: Pool of public IPs assigned dynamically
  • PAT (Port Address Translation): Many private IPs share one public IP using different ports

NAT Operation Example

Private Network (192.168.1.0/24) ← NAT Router → Internet (Public IPs)

192.168.1.10:1024 → 203.0.113.1:5000 → Web Server

First Hop Redundancy Protocol (FHRP)

Provides gateway redundancy by allowing multiple routers to share a virtual IP address.

Common FHRP Protocols

  • HSRP (Hot Standby Router Protocol): Cisco proprietary
  • VRRP (Virtual Router Redundancy Protocol): Industry standard
  • GLBP (Gateway Load Balancing Protocol): Cisco, provides load balancing

Virtual IP (VIP) Concept

Clients use the VIP as their default gateway. Active router responds to VIP traffic.

2.2 Switching Technologies and Features

Virtual Local Area Networks (VLANs)

VLAN Fundamentals

VLANs logically segment a physical network into multiple broadcast domains.

  • Reduces broadcast traffic and improves security
  • Allows flexible network design independent of physical layout
  • Each VLAN operates as a separate subnet
  • Inter-VLAN communication requires Layer 3 routing

VLAN Database and Configuration

# Creating VLANs on a switch vlan 10 name Sales vlan 20 name Engineering vlan 30 name Management # Assigning ports to VLANs interface fastethernet 0/1 switchport mode access switchport access vlan 10

Switch Virtual Interface (SVI)

Logical interface that represents a VLAN on a Layer 3 switch.

  • Allows the switch to route between VLANs
  • Provides a management interface for the VLAN
  • Can be assigned an IP address for remote management
Interface Configuration

Native VLAN

  • Untagged VLAN on trunk ports
  • Default is VLAN 1, should be changed for security
  • Must match on both ends of trunk link
  • Used for management traffic in many implementations

Voice VLAN

Special VLAN configuration for VoIP phones.

interface fastethernet 0/5 switchport mode access switchport access vlan 10 # Data VLAN switchport voice vlan 20 # Voice VLAN spanning-tree portfast # Immediate forwarding

802.1Q VLAN Tagging

Field Size Purpose
TPID 2 bytes Tag Protocol Identifier (0x8100)
Priority 3 bits Class of Service (CoS)
DEI 1 bit Drop Eligible Indicator
VLAN ID 12 bits VLAN identifier (1-4094)

Link Aggregation

Combines multiple physical links into one logical link.

  • LACP (802.3ad): Dynamic link aggregation protocol
  • Static LAG: Manually configured, no protocol negotiation
  • Provides increased bandwidth and redundancy
  • Load balancing across member links

Speed and Duplex Settings

Setting Description Best Practice
Auto Automatic negotiation Recommended for most scenarios
10/100/1000 Fixed speed Use when auto-negotiation fails
Full/Half Duplex Communication direction Full duplex for modern networks
Spanning Tree Protocol (STP)

STP Purpose and Operation

Prevents Layer 2 loops in redundant network topologies.

  • Creates loop-free topology by blocking redundant paths
  • Automatically activates backup paths when primary fails
  • Uses Bridge Protocol Data Units (BPDUs) for communication
  • Election process determines root bridge and port roles

STP Port States

State Duration Function
Disabled - Port is shut down
Blocking 20 seconds Receives BPDUs only
Listening 15 seconds Processes BPDUs
Learning 15 seconds Builds MAC table
Forwarding - Normal operation

STP Variants

  • STP (802.1D): Original, 50-second convergence
  • RSTP (802.1w): Rapid convergence, 6 seconds max
  • MSTP (802.1s): Multiple instances for VLAN groups
  • PVST+: Cisco proprietary, per-VLAN spanning tree
Maximum Transmission Unit (MTU)

MTU Fundamentals

Maximum size of a data packet that can be transmitted over a network.

  • Standard Ethernet MTU: 1500 bytes
  • Larger packets require fragmentation
  • Path MTU Discovery determines smallest MTU in path
  • Mismatched MTUs can cause connectivity issues

Jumbo Frames

Ethernet frames larger than standard 1500-byte MTU.

  • Typically 9000 bytes (9216 bytes including headers)
  • Reduces CPU overhead for large data transfers
  • Must be supported end-to-end
  • Common in storage networks and data centers
Exam Tip: Remember that all devices in the path must support jumbo frames, or fragmentation will occur.

2.3 Wireless Devices and Technologies

Wireless Channels and Frequency Management

Channel Configuration

Frequency Band Available Channels Non-Overlapping Channel Width
2.4 GHz 1-11 (US), 1-13 (EU) 1, 6, 11 20/40 MHz
5 GHz 36, 40, 44, 48, 149, 153, 157, 161 Multiple options 20/40/80/160 MHz
6 GHz 1-233 (WiFi 6E) Many options 20/40/80/160 MHz

Channel Width Considerations

  • 20 MHz: Maximum compatibility, more channels available
  • 40 MHz: Higher throughput, fewer non-overlapping channels
  • 80 MHz: Maximum throughput, limited channel availability
  • 160 MHz: WiFi 6, requires compatible devices

Regulatory Impacts (802.11h)

Dynamic Frequency Selection (DFS) and Transmit Power Control (TPC)

  • Required in 5 GHz band to avoid radar interference
  • DFS scans for radar signals before using channels
  • TPC automatically adjusts transmission power
  • May cause brief disconnections during channel changes

Band Steering

Technology that encourages dual-band devices to use optimal frequency.

  • Directs capable devices to 5 GHz band
  • Reduces congestion on 2.4 GHz
  • Uses techniques like delayed probe responses
  • May cause connection issues with some devices
Service Set Identifiers (SSIDs)

SSID Types

  • SSID: Network name broadcast to clients (up to 32 characters)
  • BSSID: MAC address of individual access point
  • ESSID: Extended SSID for multiple APs with same network name

SSID Relationship Diagram

ESSID: "Corporate_WiFi"

├── AP1 (BSSID: 00:11:22:33:44:55)

├── AP2 (BSSID: 00:11:22:33:44:66)

└── AP3 (BSSID: 00:11:22:33:44:77)

SSID Best Practices

  • Use descriptive but not revealing names
  • Avoid default manufacturer SSIDs
  • Consider hiding SSIDs for security (security through obscurity)
  • Use different SSIDs for guest and corporate networks
Wireless Network Types

Infrastructure Mode

Most common wireless deployment with central access point.

  • All communication goes through access point
  • Provides connection to wired network
  • Centralized management and security
  • Scalable with multiple APs

Ad Hoc (IBSS)

Direct device-to-device communication without infrastructure.

  • Independent Basic Service Set (IBSS)
  • No access point required
  • Limited range and scalability
  • Used for temporary connections

Mesh Networks

Multiple access points work together to provide seamless coverage.

  • Self-healing network topology
  • Automatic path optimization
  • Extended range without wired backhaul
  • Common protocols: 802.11s, proprietary solutions

Point-to-Point Links

Dedicated wireless connection between two locations.

  • High-gain directional antennas
  • Long-distance connectivity
  • Building-to-building links
  • Requires line-of-sight
Wireless Encryption and Security

WiFi Protected Access (WPA) Evolution

Standard Encryption Key Management Security Level
WEP RC4 Static keys Broken - Do not use
WPA TKIP PSK or 802.1X Legacy - Deprecated
WPA2 AES-CCMP PSK or 802.1X Good - Current standard
WPA3 AES-GCMP SAE or 802.1X Best - Latest standard

WPA3 Improvements

  • SAE (Simultaneous Authentication of Equals): Replaces PSK, resistant to offline attacks
  • Enhanced Open: Encryption even on open networks
  • 192-bit Security: For enterprise networks (WPA3-Enterprise)
  • Easy Connect: QR code-based device onboarding
Authentication Methods

Pre-Shared Key (PSK) vs. Enterprise

Method Use Case Pros Cons
PSK (Personal) Home, Small Office Simple setup, no infrastructure Shared password, no user tracking
Enterprise (802.1X) Corporate Networks Individual credentials, centralized Complex setup, requires RADIUS

Guest Networks and Captive Portals

  • Guest Networks: Isolated SSID for visitors
  • Captive Portals: Web-based authentication/acceptance
  • VLAN isolation from corporate network
  • Bandwidth limitations and time restrictions
  • Legal compliance (terms of service acceptance)
Antenna Types and Access Point Management

Antenna Patterns

  • Omnidirectional: 360-degree coverage, general purpose
  • Directional: Focused beam, point-to-point links
  • Sector: 120-degree coverage, outdoor deployments
  • Patch: High-gain directional, wall mounting

Access Point Types

Type Management Use Case Scalability
Autonomous Individual configuration Small deployments Limited
Lightweight (Thin) Controller-based Enterprise networks Highly scalable

2.4 Physical Installation Factors

Installation Locations and Infrastructure

Distribution Frame Hierarchy

  • MDF (Main Distribution Frame): Central wiring point, houses core network equipment
  • IDF (Intermediate Distribution Frame): Floor or department wiring closets
  • Hierarchical design reduces cable runs and improves organization
  • Fiber connections typically used between MDF and IDFs

Typical Distribution Frame Layout

MDF (Building Core)

├── IDF Floor 1 ←→ Horizontal Cabling to Workstations

├── IDF Floor 2 ←→ Horizontal Cabling to Workstations

└── IDF Floor 3 ←→ Horizontal Cabling to Workstations

Rack Sizing and Organization

Rack Unit (RU) Height Common Equipment
1U 1.75 inches Switches, patch panels
2U 3.5 inches Routers, servers
4U 7 inches Large switches, servers
  • Standard rack width: 19 inches
  • Common heights: 42U, 45U, 48U
  • Leave space for cable management and airflow
  • Consider weight distribution and center of gravity

Airflow Management

  • Port-side Exhaust: Hot air exits through ports (front-to-back)
  • Port-side Intake: Cool air enters through ports (back-to-front)
  • Match equipment airflow direction with rack design
  • Use blanking panels to prevent air recirculation
  • Maintain hot aisle/cold aisle configuration
Cabling Infrastructure

Patch Panels

Organized termination points for horizontal cabling.

  • 24-port and 48-port configurations most common
  • Provides organization and easy moves/adds/changes
  • Protects equipment ports from wear
  • Allows for testing and documentation

Fiber Distribution Panels

Specialized panels for fiber optic terminations.

  • Accommodate various connector types (LC, SC, ST)
  • Provide splice enclosures for fusion splicing
  • Include cable management for bend radius protection
  • Often include adapter panels for different connector types

Lockable Enclosures

  • Physical security for network equipment
  • Prevents unauthorized access to devices
  • Protects against tampering and theft
  • Required in public or semi-public areas
Exam Tip: Remember the importance of cable management - proper organization prevents outages and makes troubleshooting much easier.
Power Systems

Uninterruptible Power Supply (UPS)

Provides backup power and power conditioning.

  • Online UPS: Continuous power conversion, best protection
  • Line-Interactive: Switches to battery during outages
  • Standby: Basic protection, cheapest option
  • Provides graceful shutdown time for equipment
  • Protects against power surges and sags

Power Distribution Unit (PDU)

Distributes power to multiple devices in rack.

  • Basic PDU: Simple power strip with multiple outlets
  • Metered PDU: Displays power consumption
  • Switched PDU: Remote control of individual outlets
  • Intelligent PDU: Full monitoring and environmental sensors

Power Load Calculations

Power Planning Formula: Total Watts = Sum of all device power consumption Add 20% safety margin Consider redundancy requirements (N+1) Example: - 24-port switch: 45W - 48-port PoE switch: 740W - Router: 85W - Server: 350W Total: 1,220W + 20% = 1,464W minimum UPS capacity

Voltage Considerations

Region Standard Voltage Frequency Plug Type
North America 120V/240V 60 Hz NEMA 5-15, 5-20
Europe 230V 50 Hz CEE 7/7 (Schuko)
Data Center 208V (3-phase) 60 Hz NEMA L15-30
Environmental Factors

Temperature Management

  • Operating Range: Typically 10°C to 35°C (50°F to 95°F)
  • ASHRAE Guidelines: 18°C to 27°C (64°F to 81°F) recommended
  • Higher temperatures reduce equipment lifespan
  • Thermal monitoring and automatic shutdown protection
  • Consider heat generated by equipment when sizing HVAC

Humidity Control

  • Optimal Range: 45% to 55% relative humidity
  • Acceptable Range: 20% to 80% relative humidity
  • Low humidity increases static electricity risk
  • High humidity can cause condensation and corrosion
  • Use humidification and dehumidification systems

Fire Suppression Systems

System Type Agent Pros Cons
Water Sprinkler Water Effective, inexpensive Damages equipment
Pre-Action Water (delayed) Reduces false discharge Complex system
Clean Agent Inert gas/chemicals No equipment damage Expensive, ventilation needed
Exam Tip: Clean agent fire suppression (like FM-200 or Inergen) is preferred in data centers because it doesn't damage electronic equipment.
