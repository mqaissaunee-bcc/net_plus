5.1 Explain the Troubleshooting Methodology

The troubleshooting methodology is a systematic approach to identifying and resolving network issues. Following a structured process ensures comprehensive problem-solving and prevents overlooking critical steps.

1

Identify the Problem

  • Gather Information: Collect details about the issue from users, logs, and monitoring systems
  • Question Users: Ask specific questions about when the problem started, what changed, and symptoms observed
  • Identify Symptoms: Document all observable symptoms and error messages
  • Determine Changes: Investigate recent configuration changes, updates, or installations
  • Duplicate the Problem: Try to reproduce the issue in a controlled manner
  • Approach Multiple Problems Individually: Isolate and address each issue separately
2

Establish a Theory of Probable Cause

  • Question the Obvious: Start with simple explanations before moving to complex ones
  • Consider Multiple Approaches:
    • Top-to-Bottom OSI Model: Start at Application Layer (Layer 7) and work down
    • Bottom-to-Top OSI Model: Start at Physical Layer (Layer 1) and work up
    • Divide and Conquer: Start in the middle (Network Layer) and move up or down based on results
Layer 7 - Application
Layer 6 - Presentation
Layer 5 - Session
Layer 4 - Transport
Layer 3 - Network
Layer 2 - Data Link
Layer 1 - Physical
3

Test the Theory to Determine the Cause

  • If Theory is Confirmed: Determine the next steps to resolve the problem
  • If Theory is Not Confirmed: Establish a new theory or escalate the issue
4

Establish a Plan of Action

  • Create a detailed plan to resolve the identified problem
  • Identify potential effects of the proposed solution
  • Consider rollback procedures if the solution fails
  • Assess impact on users and business operations
5

Implement the Solution or Escalate

  • Execute the planned solution carefully
  • Monitor the implementation process
  • Escalate to appropriate personnel if necessary
  • Follow change management procedures
6

Verify Full System Functionality

  • Test the affected system thoroughly
  • Verify that the original problem is resolved
  • Ensure no new problems were introduced
  • Implement preventive measures if applicable
7

Document Findings and Actions

  • Record all findings, actions, and outcomes
  • Document lessons learned for future reference
  • Update knowledge base and procedures
  • Share information with relevant team members

5.2 Troubleshoot Common Cabling and Physical Interface Issues

Cable Issues

Incorrect Cable Types
  • Single-mode vs. Multimode Fiber:
    • Single-mode: Long distance, laser light source
    • Multimode: Short distance, LED light source
    • Cannot be mixed in the same link
  • Category Cable Mismatches:
    • Cat 5: 100 Mbps, 100 MHz
    • Cat 6: 1 Gbps, 250 MHz
    • Cat 7: 10 Gbps, 600 MHz
    • Cat 8: 25/40 Gbps, 2000 MHz
  • STP vs. UTP:
    • Shielded vs. Unshielded Twisted Pair
    • Environmental considerations
Signal Degradation
  • Crosstalk: Signal interference between wire pairs
  • Interference: EMI/RFI from external sources
  • Attenuation: Signal loss over distance
Improper Termination
  • Incorrect wiring standards (T568A vs T568B)
  • Poor connector crimping
  • Exposed wire pairs
  • Incorrect pin assignments
TX/RX Transposed
  • Transmit and Receive pairs swapped
  • Prevents proper communication
  • Common in fiber optic connections

Interface Issues

Increasing Interface Counters
  • CRC (Cyclic Redundancy Check): Indicates data corruption
  • Runts: Frames smaller than 64 bytes
  • Giants: Frames larger than 1518 bytes
  • Drops: Packets discarded due to buffer overflow
Port Status Issues
  • Error Disabled: Port automatically shut down due to errors
  • Administratively Down: Port manually disabled
  • Suspended: Port temporarily disabled by protocol
# Check interface status show interface status show interface ethernet 0/1 show interface counters errors

Hardware Issues

Power over Ethernet (PoE)
  • Power Budget Exceeded:
    • Total power draw exceeds switch capacity
    • Devices may not power on or function properly
  • Incorrect Standard:
    • PoE (802.3af): 15.4W
    • PoE+ (802.3at): 30W
    • PoE++ (802.3bt): 60W/100W
Transceiver Issues
  • Mismatch: Incompatible transceiver types
  • Signal Strength: Weak or excessive optical power
# Check PoE status show power inline show power inline interface # Check transceiver information show interface transceiver show interface dom

5.3 Troubleshoot Common Issues with Network Services

Switching Issues

Spanning Tree Protocol (STP) Issues
  • Network Loops:
    • Redundant paths without STP
    • Broadcast storms
    • MAC table instability
  • Root Bridge Selection:
    • Suboptimal root bridge placement
    • Bridge priority configuration
    • Impact on network topology
  • Port Roles:
    • Root Port: Best path to root bridge
    • Designated Port: Forwarding port on segment
    • Blocked Port: Prevents loops
  • Port States:
    • Blocking → Listening → Learning → Forwarding
    • Convergence time issues
# STP troubleshooting commands show spanning-tree show spanning-tree root show spanning-tree interface show spanning-tree blockedports
VLAN Assignment Issues
  • Incorrect VLAN membership
  • Missing VLAN configuration
  • Trunk port configuration errors
  • Native VLAN mismatches
# VLAN troubleshooting show vlan brief show interface switchport show interface trunk
Access Control Lists (ACLs)
  • Incorrect rule order
  • Implicit deny statements
  • Source/destination mismatches
  • Protocol and port specifications

Routing Issues

Route Selection Problems
  • Routing Table Issues:
    • Missing routes
    • Incorrect next hop
    • Route summarization problems
  • Default Route Problems:
    • Missing default gateway
    • Incorrect default route
    • Multiple default routes
# Routing troubleshooting show ip route show ip route summary traceroute [destination] ping [destination]

IP Address Issues

Address Pool Exhaustion
  • DHCP scope depletion
  • IP address conflicts
  • Lease time configuration
Configuration Errors
  • Incorrect Default Gateway
  • Incorrect IP Address
  • Duplicate IP Address
  • Incorrect Subnet Mask
# IP configuration troubleshooting ipconfig /all (Windows) ifconfig (Linux/Unix) ip addr show (Linux) # DHCP troubleshooting ipconfig /release ipconfig /renew ipconfig /flushdns

5.4 Troubleshoot Common Performance Issues

Network Performance Metrics

Congestion/Contention

Network congestion occurs when traffic demand exceeds available bandwidth, causing performance degradation.

  • Multiple devices competing for bandwidth
  • Collision domains in shared media
  • Buffer overflow at network devices
Bottlenecking

Performance bottlenecks occur at specific points in the network infrastructure.

  • Uplink saturation
  • CPU or memory limitations on devices
  • Single points of failure
  • Inadequate QoS implementation
Bandwidth Issues

Throughput Capacity: The actual data transfer rate achieved in practice.

  • Theoretical vs. actual bandwidth
  • Protocol overhead impact
  • Duplex mismatches
  • Cable quality and distance limitations

Network Quality Metrics

Latency

The time delay in data transmission from source to destination.

  • Propagation Delay: Physical distance limitations
  • Processing Delay: Device processing time
  • Serialization Delay: Time to place bits on wire
  • Queuing Delay: Buffer wait time
Packet Loss

Occurs when packets fail to reach their destination.

  • Buffer overflow
  • Network congestion
  • Hardware failures
  • Configuration errors
Jitter

Variation in packet delay, critical for real-time applications.

  • Network congestion fluctuations
  • Route changes
  • Queue management issues
  • Impact on VoIP and video

Wireless Performance Issues

Interference
  • Channel Overlap:
    • 2.4 GHz: Channels 1, 6, 11 (non-overlapping)
    • 5 GHz: More channels available
    • Adjacent channel interference
  • External Interference:
    • Microwave ovens
    • Bluetooth devices
    • Other wireless networks
Signal Issues
  • Signal Degradation or Loss:
    • Distance from access point
    • Physical obstructions
    • Antenna positioning
  • Insufficient Wireless Coverage:
    • Dead zones
    • Inadequate access point placement
    • Power level configuration
Client Connection Issues
  • Client Disassociation Issues:
    • Weak signal strength
    • Authentication problems
    • Power management settings
  • Roaming Misconfiguration:
    • Incorrect roaming thresholds
    • Inadequate overlap between APs
    • Different security settings

5.5 Use Appropriate Tools or Protocols to Solve Networking Issues

Software Tools

Protocol Analyzer (Wireshark)

Captures and analyzes network traffic at the packet level.

  • Real-time packet capture
  • Deep packet inspection
  • Protocol decoding
  • Performance analysis
# Wireshark filters ip.addr == 192.168.1.1 tcp.port == 80 dns http.request.method == "GET"
Command Line Tools

ping

Tests basic connectivity using ICMP echo requests.

ping 8.8.8.8 ping -t google.com (Windows continuous) ping -c 5 google.com (Linux 5 packets)

traceroute/tracert

Shows the path packets take to reach a destination.

tracert google.com (Windows) traceroute google.com (Linux/Unix) traceroute -n google.com (no DNS resolution)

nslookup

Queries DNS servers for name resolution.

nslookup google.com nslookup google.com 8.8.8.8 nslookup > set type=MX > google.com

dig

Advanced DNS lookup tool (Linux/Unix).

dig google.com dig @8.8.8.8 google.com MX dig +trace google.com

tcpdump

Command-line packet capture tool.

tcpdump -i eth0 tcpdump -i eth0 host 192.168.1.1 tcpdump -i eth0 port 80

netstat

Displays network connections and statistics.

netstat -an netstat -rn (routing table) netstat -i (interface statistics)

ip/ifconfig/ipconfig

Configure and display network interface information.

ipconfig /all (Windows) ifconfig (Linux/Unix) ip addr show (Linux) ip route show (Linux routing)

arp

Displays and modifies ARP table entries.

arp -a arp -d 192.168.1.1 (delete entry) arp -s 192.168.1.1 00:11:22:33:44:55 (static entry)
Network Mapping and Discovery

Nmap (Network Mapper)

Network discovery and security auditing tool.

nmap 192.168.1.0/24 (network scan) nmap -sS 192.168.1.1 (SYN scan) nmap -O 192.168.1.1 (OS detection) nmap -sV 192.168.1.1 (service version)

LLDP/CDP

Discovery protocols for network topology mapping.

  • LLDP: IEEE 802.1AB standard
  • CDP: Cisco proprietary protocol
  • Discovers neighbor devices
  • Provides device capabilities
show lldp neighbors show cdp neighbors detail

Speed Tester

Measures network throughput and performance.

  • Online speed testing tools
  • iperf for controlled testing
  • Bandwidth measurement

Hardware Tools

Cable Testing Tools

Toner (Tone Generator)

  • Traces cables through walls and panels
  • Tone generator and probe
  • Cable identification

Cable Tester

  • Tests cable continuity
  • Identifies wiring problems
  • Pin-to-pin mapping
  • Length measurements

Visual Fault Locator

  • Fiber optic testing tool
  • Red laser light source
  • Identifies breaks and bends
  • Connector inspection
Network Analysis Tools

Network Taps

  • Passive monitoring devices
  • Full-duplex traffic capture
  • No network impact
  • Security and troubleshooting

Wi-Fi Analyzer

  • RF spectrum analysis
  • Channel utilization
  • Signal strength mapping
  • Interference identification

Practice Scenarios

Scenario 1: Connectivity Issues

Problem: Users in VLAN 10 cannot access the internet, but users in VLAN 20 can. Both VLANs are on the same switch.

Troubleshooting Steps:

  1. Verify VLAN configuration: show vlan brief
  2. Check inter-VLAN routing: Verify SVI configuration for VLAN 10
  3. Verify routing table: show ip route
  4. Check ACLs: Look for blocking rules affecting VLAN 10
  5. Test default gateway: Ping from VLAN 10 devices

Likely Causes:

  • Missing or incorrect SVI configuration for VLAN 10
  • ACL blocking traffic from VLAN 10
  • Missing default route for VLAN 10 subnet
  • DHCP not providing correct default gateway
Scenario 2: Performance Degradation

Problem: Network performance has significantly decreased. Users report slow file transfers and web browsing. Ping times to servers have increased from 5ms to 50ms.

Troubleshooting Steps:

  1. Check interface utilization: show interface for bandwidth usage
  2. Monitor for errors: Look for CRC, drops, and collision counters
  3. Analyze traffic patterns: Use protocol analyzer to identify heavy traffic
  4. Check for broadcast storms: Monitor broadcast traffic levels
  5. Verify duplex settings: Ensure no half-duplex mismatches

Possible Solutions:

  • Implement QoS to prioritize critical traffic
  • Upgrade bandwidth on congested links
  • Fix duplex mismatches
  • Identify and limit broadcast traffic sources
  • Load balance traffic across multiple paths
Scenario 3: Wireless Connectivity Issues

Problem: Wireless clients in the conference room frequently disconnect and experience poor performance. The access point is newly installed.

Troubleshooting Steps:

  1. Check signal strength: Use Wi-Fi analyzer to measure RSSI
  2. Scan for interference: Look for overlapping channels and external interference
  3. Verify channel selection: Ensure non-overlapping channels (1, 6, 11 for 2.4GHz)
  4. Check power levels: Adjust AP transmit power if necessary
  5. Analyze client behavior: Look for roaming issues and authentication problems

Potential Solutions:

  • Change to less congested channel
  • Adjust AP placement for better coverage
  • Configure appropriate power levels
  • Enable band steering to 5GHz
  • Update client drivers and firmware
Scenario 4: DHCP Issues

Problem: New devices cannot obtain IP addresses automatically. Existing devices maintain connectivity. DHCP server appears to be running normally.

Troubleshooting Steps:

  1. Check DHCP scope: Verify available addresses in pool
  2. Monitor DHCP logs: Look for lease requests and responses
  3. Test DHCP relay: Ensure DHCP relay agent is functioning
  4. Verify network connectivity: Test connectivity between client and DHCP server
  5. Check for IP conflicts: Look for duplicate IP addresses

Common Causes:

  • DHCP scope exhaustion
  • DHCP relay agent misconfiguration
  • VLAN configuration preventing DHCP traffic
  • Firewall blocking DHCP ports (67/68)
  • Rogue DHCP server conflict
