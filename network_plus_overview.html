1.0 Networking Concepts
23%
  • 1.1 OSI Reference Model Concepts ▼
    • Layer 1 - Physical
    • Layer 2 - Data Link
    • Layer 3 - Network
    • Layer 4 - Transport
    • Layer 5 - Session
    • Layer 6 - Presentation
    • Layer 7 - Application
  • 1.2 Networking Appliances, Applications, and Functions ▼
    • Physical and virtual appliances (Router, Switch, Firewall, IDS/IPS, Load balancer, Proxy, NAS, SAN, Wireless AP/Controller)
    • Applications (Content Delivery Network)
    • Functions (VPN, QoS, Time to Live)
  • 1.3 Cloud Concepts and Connectivity Options ▼
    • Network Functions Virtualization (NFV)
    • Virtual Private Cloud (VPC)
    • Network security groups and lists
    • Cloud gateways (Internet gateway, NAT gateway)
    • Cloud connectivity options (VPN, Direct Connect)
    • Deployment models (Public, Private, Hybrid)
    • Service models (SaaS, IaaS, PaaS)
    • Scalability, Elasticity, Multitenancy
  • 1.4 Common Networking Ports, Protocols, Services, and Traffic Types ▼
    • Protocol/Port mappings (FTP 20/21, SSH 22, HTTP 80, HTTPS 443, DNS 53, DHCP 67/68, etc.)
    • Internet Protocol types (ICMP, TCP, UDP, GRE, IPSec with AH/ESP/IKE)
    • Traffic types (Unicast, Multicast, Anycast, Broadcast)
  • 1.5 Transmission Media and Transceivers ▼
    • Wireless (802.11 standards, Cellular, Satellite)
    • Wired (802.3 standards, Single-mode vs. multimode fiber, DAC cable, Coaxial cable)
    • Transceivers (Ethernet, Fibre Channel, SFP, QSFP form factors)
    • Connector types (SC, LC, ST, MPO, RJ11, RJ45, F-type, BNC)
  • 1.6 Network Topologies, Architectures, and Types ▼
    • Topologies (Mesh, Hybrid, Star/hub and spoke, Spine and leaf, Point to point)
    • Three-tier hierarchical model (Core, Distribution, Access)
    • Collapsed core architecture
    • Traffic flows (North-south, East-west)
  • 1.7 IPv4 Network Addressing ▼
    • Public vs. private addressing (APIPA, RFC1918, Loopback/localhost)
    • Subnetting (Variable Length Subnet Mask, CIDR)
    • IPv4 address classes (Class A, B, C, D, E)
  • 1.8 Evolving Use Cases for Modern Network Environments ▼
    • Software-defined networking (SDN and SD-WAN)
    • Virtual Extensible Local Area Network (VXLAN)
    • Zero Trust Architecture (ZTA)
    • Secure Access Secure Edge (SASE)/Security Service Edge (SSE)
    • Infrastructure as Code (IaC)
    • IPv6 addressing and compatibility requirements
2.0 Network Implementation
20%
  • 2.1 Characteristics of Routing Technologies ▼
    • Static routing
    • Dynamic routing (BGP, EIGRP, OSPF)
    • Route selection (Administrative distance, Prefix length, Metric)
    • Address translation (NAT, Port Address Translation)
    • First Hop Redundancy Protocol (FHRP)
    • Virtual IP (VIP) and Subinterfaces
  • 2.2 Configure Switching Technologies and Features ▼
    • Virtual Local Area Network (VLAN database, Switch Virtual Interface)
    • Interface configuration (Native VLAN, Voice VLAN, 802.1Q tagging, Link aggregation)
    • Speed and Duplex settings
    • Spanning Tree Protocol
    • Maximum Transmission Unit (MTU) and Jumbo frames
  • 2.3 Select and Configure Wireless Devices and Technologies ▼
    • Channels (Channel width, Non-overlapping channels, Regulatory impacts 802.11h)
    • Frequency options (2.4GHz, 5GHz, 6GHz, Band steering)
    • Service Set Identifier (SSID, BSSID, ESSID)
    • Network types (Mesh networks, Ad hoc, Point to point, Infrastructure)
    • Encryption (WPA2, WPA3)
    • Guest networks and Captive portals
    • Authentication (Pre-shared key vs. Enterprise)
    • Antennas (Omnidirectional vs. directional)
    • Autonomous vs. lightweight access point
  • 2.4 Important Factors of Physical Installations ▼
    • Installation implications (IDF/MDF locations, Rack size, Port-side exhaust/intake)
    • Cabling (Patch panel, Fiber distribution panel, Lockable)
    • Power (UPS, PDU, Power load, Voltage)
    • Environmental factors (Humidity, Fire suppression, Temperature)

    • Site-to-site VPN

4.0 Network Security
14%
  • 4.1 Basic Network Security Concepts ▼
    • Logical security (Encryption, Certificates, PKI, IAM, Authentication methods)
    • Physical security (Camera, Locks)
    • Deception technologies (Honeypot, Honeynet)
    • Common security terminology (Risk, Vulnerability, Exploit, Threat, CIA triad)
    • Audits and regulatory compliance (Data locality, PCI DSS, GDPR)
    • Network segmentation enforcement (IoT/IIoT, SCADA/ICS/OT, Guest, BYOD)
  • 4.2 Types of Attacks and Their Impact ▼
    • DoS/DDoS attacks
    • VLAN hopping
    • MAC flooding
    • ARP poisoning and spoofing
    • DNS poisoning and spoofing
    • Rogue devices and services (DHCP, AP)
    • Evil twin attacks
    • On-path attacks
    • Social engineering (Phishing, Dumpster diving, Shoulder surfing, Tailgating)
    • Malware
  • 4.3 Network Security Features, Defense Techniques, and Solutions ▼
    • Device hardening (Disable unused ports and services, Change default passwords)
    • Network Access Control (Port security, 802.1X, MAC filtering)
    • Key management
    • Security rules (Access Control List, URL filtering, Content filtering)
    • Zones (Trusted vs. untrusted, Screened subnet)
5.0 Network Troubleshooting
24%
  • 5.1 Troubleshooting Methodology ▼
    • Identify the problem (Gather information, Question users, Identify symptoms)
    • Establish a theory of probable cause
    • Test the theory to determine the cause
    • Establish a plan of action to resolve the problem
    • Implement the solution or escalate as necessary
    • Verify full system functionality and implement preventive measures
    • Document findings, actions, outcomes, and lessons learned
  • 5.2 Common Cabling and Physical Interface Issues ▼
    • Cable issues (Incorrect cable, Signal degradation, Crosstalk, Interference, Attenuation)
    • Interface issues (Interface counters, Port status, Error conditions)
    • Hardware issues (PoE problems, Transceiver mismatches, Signal strength)
  • 5.3 Common Issues with Network Services ▼
    • Switching issues (STP, Network loops, Incorrect VLAN assignment, ACLs)
    • Route selection (Routing table, Default routes)
    • Address pool exhaustion
    • Incorrect default gateway, IP address, subnet mask
    • Duplicate IP address issues
  • 5.4 Common Performance Issues ▼
    • Congestion/contention and Bottlenecking
    • Bandwidth and Throughput capacity
    • Latency, Packet loss, and Jitter
    • Wireless issues (Interference, Signal degradation, Insufficient coverage, Client disassociation, Roaming misconfiguration)
  • 5.5 Appropriate Tools and Protocols for Network Issues ▼
    • Software tools (Protocol analyzer, Command line tools: ping, traceroute, nslookup, tcpdump, dig, netstat, ip/ifconfig/ipconfig, arp, Nmap, LLDP/CDP, Speed tester)
    • Hardware tools (Toner, Cable tester, Taps, Wi-Fi analyzer, Visual fault locator)
    • Basic networking device commands (show mac-address-table, show route, show interface, show config, show arp, show vlan, show power)
